## ‚ùì **Clarifying Assumptions (Minimal & Critical)**

Before solving, we confirm:
- ‚úÖ Water flows **from higher to lower or equal** elevation ‚Üí `height[next] <= height[current]`.
- ‚úÖ Water can flow in **4 directions**: North, South, East, West.
- ‚úÖ Oceans border edges:  
  - **Pacific**: top row (`r=0`) and left column (`c=0`).  
  - **Atlantic**: bottom row (`r=m-1`) and right column (`c=n-1`).
- ‚úÖ We need cells that can reach **BOTH oceans** ‚Äî not just one.
- ‚úÖ Grid size: `m, n ‚â§ 200` ‚Üí DFS/BFS is safe (max 40,000 cells).

---

## 1Ô∏è‚É£ **Exact Question Summary** üéØ

> Given an `m x n` grid of heights, find all cells from which rainwater can flow **to both the Pacific Ocean and the Atlantic Ocean**, moving only to adjacent cells of **equal or lower height**.

---

## 2Ô∏è‚É£ **Key Concepts** üí°

- **üåä Water Flow Logic**: Water flows **downhill or flat** ‚Äî so from cell `(r,c)` to `(nr,nc)` if `height[nr][nc] ‚â§ height[r][c]`.
- **üîÑ Reverse Thinking (Inverted DFS)**: Instead of starting from each cell and checking if it reaches both oceans (too slow), start from **ocean borders** and see which cells water can **flow back into** (i.e., reverse direction: from ocean ‚Üí uphill).
- **‚úÖ Two Boolean Matrices**:  
  - `canReachPacific[r][c]` = true if water can flow from `(r,c)` to Pacific.  
  - `canReachAtlantic[r][c]` = true if water can flow from `(r,c)` to Atlantic.  
  ‚Üí Final answer: cells where **both are true**.
- **üß© DFS / BFS from Ocean Borders**: Start DFS from every Pacific border cell ‚Üí mark reachable cells. Repeat for Atlantic.

| Constraint | Implication |
|----------|------------|
| `1 ‚â§ m, n ‚â§ 200` | Max 40,000 cells ‚Üí DFS/BFS feasible |

| **Water flows downhill** | So reverse traversal: from ocean ‚Üí uphill (‚â• current) |

### üîç Critical Edge Cases:
- Single cell ‚Üí `[0,0]` must be in result (touches both oceans).
- Flat terrain ‚Üí entire grid may be reachable.
- Mountains blocking flow ‚Üí some regions isolated.
- All same height ‚Üí all cells reachable from both oceans.

## 5Ô∏è‚É£ **Key Insights** üß†‚ú®

- **Reverse Flow is Key**: Instead of asking ‚ÄúCan this cell reach the ocean?‚Äù, ask ‚ÄúWhich cells can flow **into** the ocean?‚Äù ‚Üí much more efficient.
- **Two Separate DFS Passes**: One for Pacific, one for Atlantic ‚Üí merge results.
- **No Backtracking Needed**: Each cell visited once per ocean ‚Üí total 2√óO(mn).
- **Memory Efficient**: Use boolean grids or bit masks ‚Äî no recursion stack explosion.

> üî• **Why reverse works**: If water flows from A ‚Üí B (A higher/equal), then in reverse, we can traverse B ‚Üí A (if A ‚â• B). So starting from ocean, we climb uphill to find source cells.

## 6Ô∏è‚É£ **Step-by-Step Algorithm** üõ†Ô∏è

### üìã Human-Readable Steps:

1. Create two boolean matrices: `pacific` and `atlantic`, size `m x n`, initialized to `false`.
2. For **Pacific Ocean**:
   - Start DFS/BFS from all cells in **top row (r=0)** and **left column (c=0)**.
   - From each, visit neighbors where `height[neighbor] >= current_height`.
   - Mark visited cells in `pacific`.
3. For **Atlantic Ocean**:
   - Start DFS/BFS from all cells in **bottom row (r=m-1)** and **right column (c=n-1)**.
   - Mark visited cells in `atlantic`.
4. Iterate over all cells: if `pacific[r][c] && atlantic[r][c]`, add `[r,c]` to result.
5. Return result.

### üñãÔ∏è Pseudocode (‚â§30 lines):

```plaintext
function pacificAtlantic(heights):
    m = rows, n = cols
    pacific = 2D bool array of size m√ón, false
    atlantic = 2D bool array of size m√ón, false

    // DFS from Pacific borders
    for r in [0, m-1]:
        for c in [0, n-1]:
            if r == 0 or c == 0:
                dfs(heights, r, c, pacific, -1)
    // DFS from Atlantic borders
    for r in [0, m-1]:
        for c in [0, n-1]:
            if r == m-1 or c == n-1:
                dfs(heights, r, c, atlantic, -1)

    result = []
    for r in 0..m-1:
        for c in 0..n-1:
            if pacific[r][c] and atlantic[r][c]:
                result.append([r, c])

    return result

function dfs(heights, r, c, visited, prevHeight):
    if out of bounds or visited[r][c] or heights[r][c] < prevHeight:
        return
    visited[r][c] = true
    for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
        dfs(heights, r+dr, c+dc, visited, heights[r][c])
```

## 7Ô∏è‚É£ **Java Solution Code** ‚òï

```java
import java.util.*;

public class Solution {
    private int[][] directions = {{0,1}, {0,-1}, {1,0}, {-1,0}};

    public List<List<Integer>> pacificAtlantic(int[][] heights) {
        int m = heights.length;
        int n = heights[0].length;

        boolean[][] pacific = new boolean[m][n];
        boolean[][] atlantic = new boolean[m][n];

        // DFS from Pacific Ocean borders (top & left)
        for (int i = 0; i < m; i++) {
            dfs(heights, i, 0, pacific, -1); // left edge
        }
        for (int j = 0; j < n; j++) {
            dfs(heights, 0, j, pacific, -1); // top edge
        }

        // DFS from Atlantic Ocean borders (bottom & right)
        for (int i = 0; i < m; i++) {
            dfs(heights, i, n - 1, atlantic, -1); // right edge
        }
        for (int j = 0; j < n; j++) {
            dfs(heights, m - 1, j, atlantic, -1); // bottom edge
        }

        // Collect cells reachable from both oceans
        List<List<Integer>> result = new ArrayList<>();
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if (pacific[i][j] && atlantic[i][j]) {
                    result.add(Arrays.asList(i, j));
                }
            }
        }

        return result;
    }

    private void dfs(int[][] heights, int r, int c, boolean[][] visited, int prevHeight) {
        // Check bounds, already visited, or too low
        if (r < 0 || r >= heights.length || c < 0 || c >= heights[0].length ||
            visited[r][c] || heights[r][c] < prevHeight) {
            return;
        }

        visited[r][c] = true;

        // Explore 4 directions
        for (int[] dir : directions) {
            int nr = r + dir[0];
            int nc = c + dir[1];
            dfs(heights, nr, nc, visited, heights[r][c]);
        }
    }

    // üß™ Optional: Main for testing
    public static void main(String[] args) {
        Solution sol = new Solution();
        int[][] test = {
            {1,2,2,3,5},
            {3,2,3,4,4},
            {2,4,5,3,1},
            {6,7,1,4,5},
            {5,1,1,2,4}
        };
        System.out.println(sol.pacificAtlantic(test));
        // Expected: [[0,4],[1,3],[1,4],[2,2],[3,0],[3,1],[4,0]]
    }
}
```

## üîü **Common Pitfalls & Optimizations** ‚ö†Ô∏è‚úÖ

### ‚ùå **Pitfalls to Avoid**:
- **Forward DFS from every cell** ‚Üí O(mn √ó mn) ‚Üí TLE.
- **Forgetting to use reverse flow logic** ‚Üí you‚Äôll miss cells.
- **Not marking ocean borders as reachable** ‚Üí e.g., `(0,0)` must be marked Pacific.
- **Using `>` instead of `>=`** ‚Üí blocks valid flat paths.
- **Stack overflow** in DFS for large grids ‚Äî but 200x200 is fine.

### ‚úÖ **Smart Optimizations**:
- Use **BFS** if you prefer iterative (avoids recursion stack).
- Combine `pacific` and `atlantic` into a single byte matrix (bit 0 = Pacific, bit 1 = Atlantic) ‚Üí saves space.
- Early termination? Not really ‚Äî need full traversal.

### üß© **Edge Case Checklist**:
- [x] Single cell  
- [x] All same height  
- [x] Strictly increasing diagonally  
- [x] Mountains blocking flow  
- [x] Zero-height cells

This problem is a **beautiful application of multi-source DFS with reverse flow logic**.

> üåä **Remember**: When water flows downhill, think **uphill from the destination** ‚Äî it‚Äôs often the key to efficiency!
